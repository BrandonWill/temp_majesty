## SMNU Panel Override — Passthrough Test

**Now generated by tooling, not hand-built.** `smnu_compiler.py` +
`cam_writer.build_cam_from_sections()` can produce a byte-perfect
passthrough CAM directly (verified in Python: real MX03 SMNU+STRT ->
`load_panel_source_from_cam()` -> `compile_panel()` ->
`build_textdata_cam()` -> `cam_reader.read_cam()` reproduces the original
bytes exactly — see SMNUResearch/test_smnu_compiler.py). What's NOT yet
verified is whether the game itself accepts this tooling-generated CAM at
load time, since byte-perfect Python roundtrip doesn't prove the engine's
loader is happy with it (e.g. CAM-level quirks, load order/scope).

- [ ] **PRIORITY — validate the currently uncommitted compiler output:**
  Generate a byte-identical MX03 passthrough CAM using
  `smnu_compiler.build_textdata_cam({"MX03": source})` (see the one-liner
  in SMNUResearch/smnu_compiler.py's module docstring / the verified demo
  from this session) and deploy it via a quest's `<CAM>` tag.
  - If no crash: confirms BOTH the override mechanism AND that
    tooling-generated CAMs load correctly — unblocks building new
    (non-passthrough) panels with this compiler.
  - If crash: something about the generated CAM structure differs from a
    real game CAM in a way Python-level roundtrip doesn't catch (e.g.
    section ordering, alignment) — needs investigation before trusting
    the compiler for anything beyond byte-perfect reproduction.
- [ ] STRT-only override test: change "Market Day" text via quest CAM
  (use `smnu_compiler.py`'s PanelSource with modified `strings`, not the
  old hand-built approach)
- [ ] Once passthrough is confirmed: test a genuinely modified panel
  (e.g. one cloned/repositioned widget via `clone_widget()`) to validate
  actual authoring, not just reproduction
